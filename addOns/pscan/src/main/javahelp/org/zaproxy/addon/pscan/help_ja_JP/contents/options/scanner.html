パッシブスキャナー

この画面では、パッシブスキャナーを設定できます。

設定オプション

フィールド 詳細 既定値 設定ファイル
スコープ内のメッセージのみをスキャン パッシブスキャンをスコープ内のメッセージに対してのみ実行するかどうかを設定します。 未選択 キー: pscans.scanOnlyInScope
値: true または false
パッシブスキャン時にファザーからのトラフィックを含める パッシブスキャンをファザーによって生成されたメッセージに対して実行するかどうかを設定します。 未選択 キー: pscans.scanFuzzerMessages
値: true または false
ルールあたりの最大アラート数 パッシブスキャンルールが発行すべき最大アラート数を設定します。 スレッド処理のためわずかに超過する場合があります。 この設定は通常、自動スキャンでのみ役立ちます。 この値を超過したスキャンルールは無効になり、新しいセッションが開始された場合は手動で有効にする必要があります。 0 (未設定) キー: pscans.maxAlertsPerRule
値: 0: 未設定、または最大アラート数
スキャン対象の最大ボディサイズ (バイト) パッシブスキャナーがスキャンするリクエストまたはレスポンスの最大ボディサイズ (バイト単位) を設定します。 これは、パッシブスキャンルールが非常に大きなリクエストやレスポンスのスキャンに過度に時間がかかる場合に使用できます。 設定されている場合、無視されたリクエストとレスポンスの数はそれぞれキー stats.pscan.reqBodyTooBig および stats.pscan.respBodyTooBig を使用して統計に記録されます。 0 (未設定) キー: pscans.maxBodySizeInBytes
値: 0: 未設定、または最大ボディサイズ (バイト)
キューをクリア メッセージをパッシブスキャンせず、パッシブスキャンキューを空にします。 現在実行中のルールは完了まで実行されますが、新しいメッセージがキューに追加された場合にのみ新しいルールが実行されます。
